# dev-skills

这个仓库用于沉淀软件工程研发流程中可复用的 Codex skills。

核心思想是：沉淀研发流程中可复用的单点能力，但尽量保持轻量高效。每个 skill 只解决一个明确问题，提供必要的流程约束、检查维度和输出格式，避免做成过重的知识库或泛化文档。

## 设计原则

- 单点能力：一个 skill 聚焦一个研发动作，例如调研、重构规划、提交前检查。
- 轻量优先：优先使用精简的 `SKILL.md`，只有确实需要时才增加 scripts、references 或 assets。
- 流程约束：把容易遗漏的步骤、风险点和验证方式固化下来。
- 可执行输出：skill 的产物应能直接进入下一步研发流程，而不是停留在泛泛建议。
- 可组合：skills 之间能串成研发流程，而不是互相重叠。

## Skills

### `prepare-commit`

提交准备技能。用于 review staged 或 unstaged changes，并在用户明确要求提交时整理变更、只 stage 相关文件、生成提交信息、完成 commit 并报告最终工作区状态。

### `refactor-plan`

重构规划技能。用于在重构前明确目标、行为保护、风险点、执行步骤和完成标准，强调小步推进、行为保持、机械改动和语义改动分离。

### `research-brief`

调研简报技能。用于从粗略想法出发补充技术背景、业界实践、替代方案和盲点，产出可进入需求分析或技术方案设计的输入。

### `codebase-orientation`

代码库导向技能。用于在既有仓库里做设计、计划、调试或实现前，先整理技术栈、运行验证命令、模块职责、入口路径、数据流、依赖集成、测试方式和风险边界。

### `bug-reproduction`

Bug 复现技能。用于修复前先明确预期与实际行为、找到真实入口、构造最小复现、收集日志/网络/数据/状态证据，并区分已确认事实、假设和下一步修复方向。

### `change-impact-analysis`

影响面分析技能。用于在设计、计划、实现、重构或提交前分析某个具体改动的直接/间接影响，识别受影响模块、接口契约、数据/配置、测试范围、兼容风险和后续技能输入。

### `write-prd`

PRD 沉淀技能。用于把已经讨论清楚或基本成型的需求内容整理成产品需求文档，明确目标、范围、用户场景、功能需求、非功能需求、验收标准、风险和后续设计输入。

### `write-trd`

TRD 沉淀技能。用于把 PRD 或已明确的产品需求转成技术方案文档，明确架构边界、接口契约、数据模型、状态流转、安全、可观测性、兼容迁移、测试策略和执行计划输入。

### `write-execution-plan`

执行计划沉淀技能。用于把 TRD 或明确的技术方案转成实施 DAG、关键路径、风险优先级、验证节点和 subagent 并发计划，并在适合并发时为每个 subagent 生成单独的轻量执行计划。

### `implement-plan`

计划实现技能。用于按执行计划逐节点实现，先选择 TDD、回归测试、特征测试、已有覆盖或手工验证等行为保护模式，再小步开发、合并 subagent 产物、阶段验证并更新进度。

### `skill-retrospective`

技能复盘技能。用于根据真实任务反馈改进已有 skills，定位触发、流程、handoff、输出、验证或命名问题，做最小修订并运行结构校验。

## Installation

本仓库中的 skill 可以通过软链安装到 Codex：

```bash
ln -sfn /path/to/dev-skills/skills/<skill-name> ~/.codex/skills/<skill-name>
```

安装或更新后，重启 Codex 以重新加载 skills。
